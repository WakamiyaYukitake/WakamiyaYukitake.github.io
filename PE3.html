テーマ案１：都市空間における氾濫流挙動の解析 ― SPH法による構造物配置の影響評価

【概要】
本テーマでは、建物の個数・フットプリント（建物の占める面積）・階数（高さ）・配置を決め、その配置が河川氾濫（ダムブレイク様式）時の流速や浸水深などにどのように影響するかを、粒子法であるSPH法を用いたGUIで構築されたDesignSPHysicsというソフトウェアを使って解析します。プログラミングの知識は必須ではありません。既定の構造物配置を基準に、配置改良案を作成し、都市構造の変化が氾濫流に与える影響を可視化・評価することを目標とします。

目的

  • 既定配置を基準に、総延べ床面積などの条件を満たす設計案を複数作成し、河川越流に対して避難所周辺の最大流速・浸水深・到達時間がどのように変化するかを定量的に比較します。
  • 配置改良による効果が生じる流体の流れ（流路誘導、分流、局所加速、渦の生成など）を可視化します。
  • 改良案の実務的意味（避難確保、建物の配置効率など）を簡潔な提言としてまとめられるレベルまで検証します。

条件設定

都市形態指標を定義・計算し、同一流入量条件で比較します。DesignSPHysicsではダムブレイク方式を用います。粒子間距離を0.01mとし、解析スケールを1/20とします。

解析手順

  1. DesignSPHysicsの準備（FreeCAD / ParaView / DualSPHysics のセットアップ）
  2. 「Pre-processing」→「New Case」を選択し解析領域を定義
  3. モデルを「Save」で保存 → 「Gencase」で粒子確認 → 「Run」で解析
  4. 「PartVTK」でvtkファイル化し、ParaViewで可視化

テーマ案２：PLATEAUデータを用いた河川氾濫解析の実施と検証

【概要】
私が現在行っている研究と全く同じ手法（DualSPHysics）を使い、国土交通省の3D都市モデル「PLATEAU」から別の地域を選び、実際に河川氾濫解析を行います。また、解析で設定するパラメータが結果にどれほど影響するかを検証し、「この設定は本当に正しかったのか？」といった、研究の妥当性を評価するプロセスにも挑戦します。

目的

  • PLATEAUデータを用いて、任意の地域（例：学生の出身地）における氾濫シミュレーションを実施し、研究手法の汎用性を確認します。
  • 流入速度、動粘性係数、粒子間距離などの主要パラメータが、解析結果（浸水域、最大浸水深など）に与える影響を定量的に評価する「感度分析」を行います。
  • 元の研究で行われた仮定の妥当性を批判的に考察し、より現実に近いシミュレーションを行うための改善点を提案します。

条件設定

対象地域はPLATEAUデータが公開されている都市から自由に選択。比較のため、堤防の決壊幅や解析時間などの基本的な設定は元の研究に準拠しつつ、検証したいパラメータのみを系統的に変化させます。

解析手順

  1. PLATEAUから対象地域のデータをダウンロードし、UnityやBlenderを用いて解析用のSTL形式に変換します。
  2. DualSPHysicsのXMLファイルで、計算領域、境界条件、検証用パラメータを設定します。
  3. シミュレーションを実行し、計算結果（VTKファイル）を出力します。
  4. ParaViewを用いて結果を可視化し、パラメータの違いによる結果の変化を比較・分析します。

テーマ案３：解析結果を活用した新しい防災情報可視化システムの開発

【概要】
シミュレーションで得られた「建物にかかる力」や「浸水深」といった物理的な解析結果と、PLATEAUが持つ「建物の構造種別（木造、鉄筋コンクリート造など）」や「築年数」といった属性情報を組み合わせ、「どの建物が倒壊しやすいか」といった、より詳細なリスクを評価・可視化するシステムの開発に挑戦します。

目的

  • 物理シミュレーションの結果と、都市の属性データを統合し、建物ごとの脆弱性を評価する新たなリスク指標を作成します。
  • Pythonなどを用いてデータ処理・分析を行い、各建物にリスクスコアを付与する手法を構築します。
  • 算出したリスクを、Web上で3D都市モデルを閲覧できる**「PLATEAU View」**に重畳表示し、インタラクティブな可視化を試みます。

開発アプローチ

解析結果から得られる「建物ごとの最大流体力」と、PLATEAUの属性情報から得られる「構造上の脆弱性（例：木造は脆弱性が高い）」を掛け合わせることで、独自の「倒壊危険度スコア」を算出します。

想定される活動内容

  1. DualSPHysicsの解析結果から、建物ごとの最大作用力データを抽出します。
  2. PLATEAUのCityGMLファイルから、Pythonなどを用いて建物ごとの構造種別や築年数データを抽出します。
  3. 両データを建物のIDをキーとして結合し、独自の計算式で各建物のリスクスコアを算出します。
  4. 算出したスコアに応じて建物を色分けしたデータセット（GeoJSONなど）を作成し、「PLATEAU View」で表示させることを目指します。
